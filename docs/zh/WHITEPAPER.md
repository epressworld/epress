# epress world: 一个去中心化的社交和内容网络

**版本 1.0**

---

### 摘要

本文提出了一种完全点对点的电子协议，旨在构建一个去中心化的社交和内容网络，该协议旨在从根本上解决当前中心化网络在数据主权、信息信任和价值沉淀上所面临的结构性困境。其核心架构由用户通过加密密钥完全控制的主权节点构成。节点间的交互通过对内容哈希进行签名，创造出将创作者身份、内容与时间永久绑定的、不可否认的“言论事实”。这些经过签名验证的事实共同构建了一个全球统一、可公开追溯的“思想图谱”。这种范式为构建一个创作者真正拥有其数字资产、贡献可被客观衡量、信誉价值得以跨平台沉淀和携带的全新价值互联网，奠定了坚实的基础。

### 1. 引言：当前互联网的结构性困境

自诞生以来，互联网已演变为全球信息交换与社会协作的核心基础设施。其当前的主导架构，建立在中心化服务器与客户端的模式之上，极大地促进了信息的传播与连接的便利性。然而，随着网络生态的成熟，这种中心化架构的内在局限性也日益凸显，并形成了三个难以调和的结构性困境。

**1.1 数据主权的中心化**

在这种主导架构下，用户虽然是内容的创造者，却并非其数字资产的最终所有者。用户发表的文字、图片、视频，以及建立的社交网络关系，都作为数据存储在服务提供商所拥有的服务器上。在这种关系中，用户更像是平台的“租客”，而非其数字家园的“房主”。

这意味着，一个拥有数百万关注者的创作者，其账号和长年累积的数字资产，可能因为一次平台政策的摇摆、一次算法的错误判断，甚至一次恶意的商业竞争，而在旦夕之间被冻结或清除。其身份、内容和社交图谱等核心数字资产，都与特定平台强绑定，无法自由迁移，并始终暴露在平台规则的系统性风险之下。

**1.2 信任的缺失与信息的确权**

当前网络协议在设计上缺少一个原生的信任层。信息本身与其来源之间，缺乏一种可被密码学公开验证的绑定关系。这种架构性的信任缺失，为虚假信息、身份冒用等问题的滋生提供了土壤，进而可能让互联网内容陷入劣币驱逐良币的恶性循环。

与此同时，用户和开发者被迫将信任寄托于平台运营商的判断，而非依赖于可验证的客观事实。这大大增加了整个网络的信息交互成本[^1]，并降低了信噪比。

**1.3 价值的孤岛化与流失**

用户的声誉、影响力与其所贡献内容的价值，被困于一个个独立的平台“孤岛”之中。一个在平台A上拥有巨大影响力的创作者，当他迁移到平台B时，其积累的信誉和社交关系几乎无法随之迁移，必须从零开始。

这意味着，一位在某个专业领域（如科学、艺术）持续贡献高质量内容并建立起崇高声望的专家，其信誉和影响力几乎无法被带出平台。当他希望在一个新的、更专注的社区里发言时，他仍需从一个“无名之辈”重新开始，他过去的贡献在新的环境中无法成为其信誉的直接证明。这种价值的孤岛化，从根本上阻碍了一个真正以创作者为中心的、价值可以自由流通和累积的互联网经济的形成。

### 2. epress 的世界：一个价值可沉淀的互联网

面对前述的结构性困境，我们需要的不是对现有平台的修补，而是一场架构层面的范式转移。epress 的核心思路，建立在两大原则之上：**个体主权**，即权利回归个人；与**可验证通信**，即交互回归事实。

基于这两大原则，一个与当前互联网形态截然不同的世界得以浮现。

**2.1 永不丢失的数字资产**

想象一下，您的数字生活如同现实生活中的搬家。无论您从一个城市搬到另一个城市，您的身份、您的藏书、您的朋友名录，都理应是可随身携带的私有财产。

在 epress 的世界里，这一愿景成为现实。您的数字身份（由加密钱包控制）、您创作的所有内容、您建立的每一个社交连接，都将是真正属于您的个人资产，而非某个公司的负债。您可以随时更换您的节点服务商（如同选择不同的电信运营商），甚至将节点部署在您家里的电脑上，而您的所有数字资产都将无缝跟随您。平台“倒闭”或“封禁”将不再意味着您数字生命的终结。

**2.2 可积累的“数字信誉”**

在 epress 的世界里，每一份经过签名的内容，都拥有一个独一无二、永不改变的地址（其内容哈希），如同一篇公开发表的论文拥有其DOI（数字对象唯一标识符）。

这使得“引用”成为一种可被精确追踪和计量的行为。一个创作者的真正价值，不再是某个平台上一瞬即逝的“流量”或虚浮的“粉丝数”，而是其思想和作品在整个网络中被引用、被讨论、被再创作的客观记录。

这份信誉与您的主权身份永久绑定，将穿越时间，不断积累，成为您最宝贵的、真正属于您自己的数字资产。在这里，贡献终于变得可衡量，价值终于得以沉淀。

**2.3 开放的创新生态**

当社交图谱和内容记录从一个个封闭的“公司内网”，变成一个开放、可信的公共数据层时，创新的浪潮将被彻底释放。任何人，无论大小公司还是独立开发者，都可以基于这套开放协议，去创造此前无法想象的应用，而无需任何人的许可。

例如：
*   **思想传播的“数字考古学”**
    由于所有内容的来源和引用关系都可被验证和追溯，研究人员第一次可以像考古学家一样，精确地绘制出任何一个观点、理论或“模因”（Meme）在网络中诞生、演化和传播的全貌。

*   **可组合的、以用户为中心的社交算法**
    在当前平台，用户只能被动接受由平台提供的单一、不透明的推荐算法。而在 epress 的开放生态中，算法可以作为一种独立的服务存在。用户将可以自由选择、组合甚至自己编写算法，来决定自己想看到一个怎样的世界——一个“只看我最信任的人”的算法，一个“最大化新奇与意外”的算法，或一个“只关注某个垂直专业领域”的算法。算法的权力，将从平台回归到用户手中。

*   **去中心化的贡献证明（Proof of Contribution）系统**
    基于可验证的引用网络，开发者可以为任何垂直领域构建专门的信誉系统。例如，一个服务于科学界的系统，可以自动计算一篇论文（一个`contentHash`）在整个 `epress` 网络中被后继研究引用的“影响力因子”；一个服务于艺术界的系统，则可以清晰地展示一个原创作品是如何启发了一系列的衍生创作。这种客观、可验证的贡献证明，将比任何主观的“点赞”都更具价值。

### 3. epress world protocol (EWP)

epress world protocol (以下简称EWP) 是我们为实现上述愿景而提出来的一个点对点的社交网络协议。它定义了一套完整的、基于密码学验证的交互规则，用于规范节点间的身份识别、连接管理和内容分发。EWP 的核心原则是“不信任，去验证”，确保网络中的每一次重要交互都有据可查、不可否认，从而构建一个无需中心化仲裁的可信网络。

**3.1 协议的核心构成**

EWP 协议建立在三个核心概念之上，它们共同构成了 epress 网络的基础：

*   **主权节点 (Node):** 网络的基础单元，一个由用户通过其以太坊地址完全拥有的软件实例，它通常表现为一个公开的个人网站。每个节点都拥有其独一无二的加密身份和一份可供外界查询的公开档案。

*   **可验证的发布 (Publication):** 网络中信息流动的基本单位。EWP 将“发布”明确区分为两种状态：
    1.  **本地发布**: 内容仅存在于创作者自己的节点上，对所有访问者可见，但所有者可随时修改或删除。这为创作者提供了自由创作和编辑的私有空间。
    2.  **签名发布**: 创作者使用其以太坊私钥，对内容的哈希（数字指纹）进行签名。这一行为创造了一个不可篡改的“言论事实”，将创作者的身份、内容本身和发布时间永久绑定。只有经过签名的发布，才能进入 EWP 的分发流程。

*   **共识连接 (Connection):** 节点之间建立的、用于信息流动的单向“关注”关系。与传统社交网络不同，EWP 中的每一次连接都必须通过一次加密“握手”来达成双方共识，确保连接的建立是双方知情且同意的结果。

**3.2 核心协议流程**

要理解上述架构在实践中的运作方式，我们首先需要建立一个全新的世界观：在 `epress` 网络中，每一个“节点”不仅是一个个人网站，更是一个独立的、活跃的“**社交个体**”，它代表了其所有者在网络中的完整存在。因此，当一个节点“关注”另一个节点时，并非是网站之间的技术链接，而是**节点所有者之间，通过其主权节点为代理，建立社会性连接并订阅对方签名发布**的意愿表达。

EWP 作为一个基于 HTTP 的应用层协议，通过一组定义清晰的 RESTful API 来协调节点间的复杂交互。所有流程都围绕着“可验证性”这一核心原则设计，确保操作的安全与高效。

#### **3.2.1 节点身份发现**

在去中心化网络中，任何交互都始于身份的确认。一个节点需要一种标准化的方式来查询另一个节点的公开档案。为此，EWP 规定：
*   任何一个 epress 节点，都必须在其对外服务的根 URL 下，实现一个标准的 `/ewp/profile` 端点。
*   通过对该端点发起一个简单的 `GET` 请求（例如 `GET https://bob.blog/ewp/profile`），任何节点都可以获取到目标节点的公开档案，其中包括节点的以太坊地址、名称、简介等核心信息。
    ```json
    {
      "address": "0x...",      // 节点的以太坊地址，是其唯一身份标识
      "title": "节点标题",      // 节点的公开名称或标题
      "url": "https://...",     // 节点可公开访问的 URL 地址
      "description": "节点简介" // 一段关于该节点的简短描述
    }
    ```

这个机制是后续所有交互（如建立连接）的基础。

#### **3.2.2 连接管理**

*   **建立连接 (Following)**
    当一个节点（关注者）希望关注另一个节点（被关注者）时，必须经过一个同步确认流程，以防止单方面的数据不一致。
    1.  **意向签名**: 关注者对一份遵循 EIP-712[^2] 标准的、包含双方身份信息的“关注意向”数据进行签名。
        ```json
        {
          "followeeAddress": "0xAddressOfFollowee", // 被关注者的以太坊地址
          "followeeUrl": "https://followee.node",   // 被关注者的节点 URL
          "followerUrl": "https://follower.node",   // 关注者的节点 URL
          "timestamp": 1678886400                 // 防止重放攻击的时间戳
        }
        ```
    2.  **提交与验证**: 签名后的意向被提交给**被关注者**的节点。被关注者节点在本地验证签名的有效性。
    3.  **同步确认**: 验证通过后，被关注者节点并**不立即**写入数据库，而是向**关注者**节点的 `/ewp/connections` 端点发起一个同步的 `POST` 请求，请求对方确认并创建连接。
    4.  **双方写入**: 关注者节点在收到确认请求、验证无误后，首先在自己的数据库中创建连接记录，并返回成功响应。被关注者节点在收到这个成功响应后，才最终在自己的数据库中创建记录。
        双方数据库中创建的连接表示例（以 Alice 关注 Bob 为例）:
        ```json
        # Alice 节点的数据库记录:
        {
          "follower_address": "self",
          "followee_address": "0xAddressOfBob",
          "created_at": "2025-09-12T10:30:00Z"
        }

        # Bob 节点的数据库记录:
        {
          "follower_address": "0xAddressOfAlice",
          "followee_address": "self",
          "created_at": "2025-09-12T10:30:00Z"
        }
        ```
    这个严谨的“两阶段提交”确保了只有在双方都成功记录后，连接才算真正建立。

*   **断开连接 (Unfollowing)**
    断开连接同样是一个需要签名的操作。发起方（关注者）签名一条“断开连接”的消息，并通过 `DELETE` 请求通告给对方节点。双方在验证签名后，各自删除本地的连接记录，从而完成解绑。
    ```json
    {
      "followeeAddress": "0xAddressOfFollowee", // 被关注者的以太坊地址
      "followerAddress": "0xAddressOfFollower", // 关注者的以太坊地址
      "timestamp": 1678886400                 // 防止重放攻击的时间戳
    }
    ```

#### **3.2.3 内容签名与分发**

EWP 的内容分发机制旨在实现高效、可验证的信息复制。

*   **内容签名 (Signing)**
    如前所述，当创作者决定将一篇本地内容分享到网络时，他们需要对其内容的 SHA-256 哈希值进行签名。这个签名与内容哈希、发布者地址、时间戳一同构成了一份不可变的“发布证明”。在协议中，这份证明被定义为一个 EIP-712 结构化数据，其主类型 (PrimaryType) 为 “ContentSignature”。
    ```json
    {
      "contentHash": "0xHashOfContent...",      // 内容的 SHA-256 哈希
      "publisherAddress": "0xAddressOfPublisher", // 发布者的以太坊地址
      "timestamp": 1678886400                  // 内容发布时间戳
    }
    ```

*   **内容分发 (Distribution)**
    为兼顾性能与安全，EWP 采用一种高效的“通知-拉取 (Notify-Pull)”模型。
    1.  **发送通知**: 发布者节点会向其所有关注者节点的 `/ewp/replications` 端点发送一个轻量级的 `POST` **通知**。该通知中仅包含“发布证明”（即签名数据），而不包含内容原文。
        其请求载荷 (payload) 的结构如下：
        ```json
        {
          "typedData": {
            "primaryType": "ContentSignature",
            "message": {
              "contentHash": "0xHashOfContent...",
              "publisherAddress": "0xAddressOfPublisher",
              "timestamp": 1678886400
            },
            ... // domain, types 等其他 EIP-712 字段
          },
          "signature": "0x..." // 用户对 typedData 的签名结果
        }
        ```
    2.  **验证与拉取**: 关注者节点收到通知后，首先验证签名的真实性，并检查发布者是否在自己的关注列表中。验证通过后，关注者节点会根据通知中的内容哈希，通过一个标准的 `GET /ewp/contents/{hash}` 请求，主动向发布者节点**拉取**完整的原始内容。

        该接口的响应根据内容类型的不同而变化，接收方需要通过解析响应头来正确处理。

        **示例 1：获取 Markdown 文本内容**
        这是网络中最核心的文本内容交换格式。响应体直接是 UTF-8 编码的纯 Markdown 文本。
        ```http
        HTTP/1.1 200 OK
        Content-Type: text/markdown; charset=utf-8
        Content-Length: 528

        # 这是一级标题

        这是正文段落，epress 网络中的主要文本信息都以这种纯粹的 Markdown 格式流通。

        * 列表项一
        * 列表项二
        ```

        **示例 2：获取图片等任意文件内容**
        对于文件类型，响应头中会包含更丰富的元数据，如文件名和文件描述。
        ```http
        HTTP/1.1 200 OK
        Content-Type: image/jpeg
        Content-Length: 213886
        Content-Disposition: inline; filename="awesome-photo.jpg"
        Content-Description: A%20photo%20of%20an%20awesome%20sunset.

        [...图片的二进制数据流...]
        ```
    3.  **最终校验与存储**: 在获取原文后，关注者节点会再次计算其哈希值，与“发布证明”中的哈希进行最终比对。只有完全一致，内容才会被存储为副本。

这个机制巧妙地将重量级的内容传输变为接收方的按需操作，极大地降低了发布者的广播负担，同时通过每一步的密码学校验，确保了内容在传输过程中的完整性和真实性。

**3.3 协议接口总览：一个标准化的节点**

通过上述流程可以看出，EWP 协议的核心思想，是为网络中的每一个节点赋予一套标准化的能力。任何一个 epress 节点，本质上都是一个独立、对等、且具备以下标准 API 接口的服务器：

*   **身份能力 (Identity)**
    *   `GET /ewp/profile`: 用于向外界提供节点的公开身份信息。
    *   `GET /ewp/avatar`: 用于提供节点的头像。
    *   `POST /ewp/nodes/updates`: 用于接收来自其他节点的 Profile 更新通告，确保节点信息（如 URL）的最终一致性。

*   **连接能力 (Connection)**
    *   `POST /ewp/connections`: 用于接收并确认一个“关注”请求。
    *   `DELETE /ewp/connections`: 用于接收并处理一个“取消关注”的通知。

*   **内容交换能力 (Content Exchange)**
    *   `POST /ewp/replications` (推送): 用于接收来自其关注对象的“新内容发布”**通知**。这是内容分发的主要方式。
    *   `GET /ewp/publications` (拉取): 用于对外提供**发布历史记录**的查询。主要用于节点在长时间离线后，重新上线时主动向其关注的节点查询错过的历史内容，是一种重要的“拉取式补全”机制。
    *   `GET /ewp/contents/{hash}`: 用于向外界提供已签名内容的原始内容下载。

所以，任何人都可以基于这个协议，使用任意编程语言开发和运行符合此协议的节点，而无数个这样的节点之间相互连接，信息在这些之间自由流动就构成了“epress world”——一个数据主权得以保证，海量数据历史来源可追溯，价值不再孤岛和流失的开放可验证的去中心化社交和内容网络。

### 4. 展望：一个可计算的信誉经济

一个全社会参与的 epress world 蕴藏着巨大想象力。EWP 协议提供的技术基石，将催生出当前互联网无法想象的全新应用与经济形态，它将互联网从一个单纯的“信息互联网”，转变为一个“**价值互联网**”。在这个网络中，思想的价值、个体的信誉第一次变得可计算、可累积、可携带。

**4.1 思想的价值发现：去中心化的“佩奇排名”**

epress world 网络在本质上是一个由“主权身份”为节点、“签名发布”为内容、“引用关系”为连接的有向图，这是一个可被数学方法分析的、全球统一的“思想图谱”。基于这一特性，开发者可以创造出去中心化的“思想搜索引擎”。一个观点（由其`contentHash`代表）的价值，将不再由平台的算法所主导，而是可以通过类似 Google PageRank 的算法，根据引用它的其他节点的信誉和数量来客观地计算。这将使得真正有价值的、深刻的思想，能够从信息的噪音中脱颖而出。

**4.2 客观的信誉衡量：可验证的贡献证明**

由于所有贡献（签名发布）都与一个永恒的身份绑定且不可篡改，一个参与者的完整贡献历史是公开、可审计的。这一特性使得构建专门的、客观的贡献证明系统成为可能。例如，在科学界，一个科学家的学术影响力可以不依赖于任何期刊，而是通过其研究成果在 `epress world` 网络中被同行的引用、讨论和验证来直接衡量。这种信誉将成为个人最有价值的、可随身携带的无形资产。

**4.3 开放的社会学实验室：思想传播的“数字考古”**

epress world 网络天然是一个由无数可验证的、带有时间戳的通信事件组成的高保真数据库。这为社会科学家和数据研究者提供了前所未有的、研究人类思想与文化如何真实传播的无偏见“培养皿”。他们可以精确地绘制出任何一个观点或文化模因（Meme）的诞生、变异、传播和消亡的全过程，而无需再依赖于被中心化平台过滤和加工后的“二手数据”。

### 5. 结论

我们在此提出了一种完全点对点的、用于内容发布与社交连接的电子协议。该系统基于密码学证明而非信任，允许任意两个节点在无需中心化服务的情况下，直接进行可验证的交互，为解决当前互联网目前遇到的核心困境提供了全新的范式。

这种由简单的基础组件构成的稳固结构，其意义超越了信息传递本身。它为思想的价值和个体的信誉提供了一个客观、可计算的基础。当贡献变得可衡量，一个创作者的信誉便能穿越平台的壁垒，成为其真正拥有的、可随身携带的数字资产。

这为构建一个其历史不可篡改、价值得以沉淀的全球性社交和内容网络，奠定了坚实的技术基石。
